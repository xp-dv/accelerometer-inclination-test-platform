This table lists the available commands, their corresponding codes, arguments, and descriptions. These commands control the system\textquotesingle{}s motion, profiles, and tests.\hypertarget{_command__table_Command}{}\doxysection{\texorpdfstring{Syntax}{Syntax}}\label{_command__table_Command}
Each instruction will follow the following format\+:

{\ttfamily \{code\texorpdfstring{$\vert$}{|}arg1 arg2 ... argN\}}

{\ttfamily \texorpdfstring{$<$}{<}code\texorpdfstring{$>$}{>}}\+: Required instruction code within the range (0-\/999). ~\newline
 {\ttfamily \mbox{[}arg1\mbox{]}, \mbox{[}arg2\mbox{]}, \mbox{[}…\mbox{]}}\+: Optional arguments. The number of arguments and range of accepted values for each argument will depend on the instruction code given. ~\newline
 \begin{quote}
Ex\+: Some instructions have no arguments, others require multiple. \end{quote}
{\ttfamily ‘\{’}\+: All instructions must begin with the start indicator. ~\newline
 {\ttfamily ‘\}’}\+: All instructions must end with the end terminator. ~\newline
 {\ttfamily ‘\texorpdfstring{$\vert$}{|}’}\+: The first argument (if applicable) must be separated from the instruction code with the instruction separator. ~\newline
 {\ttfamily ‘ ’}\+: All subsequent arguments must be separated by the argument separator. ~\newline
 All numbers must be represented in decimal.

After an instruction is carried out, the received instruction will be echoed by the device in the following format\+:
